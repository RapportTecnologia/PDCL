Este projeto implementa a metodologia PDCL (Plan, Do, Check, Logs) para desenvolvimento de software, um ciclo iterativo que garante qualidade e rastreabilidade em todas as fases do desenvolvimento. Na fase Plan, você deve planejar detalhadamente todas as funcionalidades, definir requisitos claros, estabelecer critérios de aceite e documentar decisões de arquitetura antes de iniciar qualquer implementação. Na fase Do, execute o desenvolvimento seguindo rigorosamente o planejamento, implemente as funcionalidades conforme especificado, mantenha código limpo e bem documentado, e registre todas as atividades relevantes nos logs específicos. Na fase Check, realize testes completos e sistemáticos de toda a aplicação, verifique se os requisitos foram atendidos, analise os logs gerados para identificar possíveis problemas, e valide a qualidade do código produzido através de revisões e testes automatizados. Na fase Logs, mantenha registros detalhados de todas as atividades, decisões, problemas encontrados e soluções aplicadas, garantindo rastreabilidade completa e permitindo análise futura para melhoria contínua. Todo o código produzido deve seguir rigorosamente esta metodologia, com cada fase sendo documentada e validada antes de prosseguir para a próxima, assegurando que o desenvolvimento seja sistemático, controlado e de alta qualidade.
